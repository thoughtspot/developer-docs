ThoughtSpot provides an official Python SDK for REST API v2.0 â€” an async-first, fully-typed client generated directly from the ThoughtSpot OpenAPI specification. The SDK wraps every REST API v2.0 endpoint into a typed Python method and supports both asynchronous and synchronous invocation, transparent token refresh, automatic retries with exponential backoff, server-sent event (SSE) streaming, file uploads and downloads, and typed exception handling.

== Quick start

// SOURCE: README â€” Usage section, first example block

Create a `Configuration`, pass it to `ThoughtSpotRestApi`, and start making calls:

[source,python]
----
import asyncio
from thoughtspot_rest_api_sdk import Configuration, ThoughtSpotRestApi

BASE_URL = "https://your-cluster.thoughtspot.cloud"  # <1>

async def main():
    config = Configuration(host=BASE_URL, access_token="YOUR_BEARER_TOKEN")  # <2>

    # Pass Configuration directly; the client manages its own connection pool.
    # To share one pool across several API classes, build an ApiClient
    # explicitly and pass that instead â€” see Per-tag API classes.
    async with ThoughtSpotRestApi(configuration=config) as client:

        # Get the current user
        user = await client.get_current_user_info()
        print(user.name)

        # Search users â€” pass a dict or a typed request model
        users = await client.search_users({"record_offset": 0, "record_size": 10})
        for u in users:
            print(u.name)

asyncio.run(main())
----
<1> Replace with your ThoughtSpot cluster URL.
<2> Replace with a valid bearer token. For production use, see xref:#authentication[Authentication] for token provider options.

== Authentication

// SOURCE: README â€” Authentication section (all three modes confirmed)
// SOURCE: Design doc Section 5.1, C11/C12/C13

The SDK accepts three authentication modes, all set on `Configuration.access_token`:

[width="100%",cols="1,2,3"]
[options="header"]
|====
|Mode |When to use |How to configure
|Static token |Short-lived scripts, testing |Pass the bearer token string directly
|Built-in token provider |Production â€” password or trusted auth |Use `ThoughtSpotTokenProvider`; mints and refreshes automatically
|Custom callable |Custom identity provider or token store |Pass a sync or async function that returns a token string
|====

=== Option 1 â€” Static bearer token

// SOURCE: README â€” "1. Static token â€” when you already hold a bearer token"

[source,python]
----
from thoughtspot_rest_api_sdk import Configuration, ThoughtSpotRestApi

config = Configuration(host=BASE_URL, access_token="YOUR_BEARER_TOKEN")
----

=== Option 2 â€” Built-in token provider (recommended for production)

// SOURCE: README â€” "2. Built-in provider â€” ThoughtSpotTokenProvider mints a token from /auth/token/full"
// The provider caches the token and only re-mints when near expiry.
// Collapses concurrent refreshes into a single call.
// Supply exactly one of password (basic auth) or secret_key (trusted authentication).
// TODO: [WRITER] Confirm the exact module import path for ThoughtSpotTokenProvider with the SDK team.
//       README shows: from thoughtspot_rest_api_sdk import Configuration, ThoughtSpotTokenProvider
//       __init__.py confirms ThoughtSpotTokenProvider is exported from the top-level package.

`ThoughtSpotTokenProvider` mints a token from `/api/rest/2.0/auth/token/full` and refreshes it transparently. Supply exactly one of `password` (basic auth) or `secret_key` (trusted authentication):

[source,python]
----
from thoughtspot_rest_api_sdk import Configuration, ThoughtSpotRestApi, ThoughtSpotTokenProvider

BASE_URL = "https://your-cluster.thoughtspot.cloud"

# Basic auth â€” supply password
provider = ThoughtSpotTokenProvider(BASE_URL, "USERNAME", password="PASSWORD")
config = Configuration(host=BASE_URL, access_token=provider)

async with ThoughtSpotRestApi(configuration=config) as client:
    user = await client.get_current_user_info()
    print(user.name)
----

For trusted authentication, supply `secret_key` instead of `password`:

[source,python]
----
provider = ThoughtSpotTokenProvider(BASE_URL, "USERNAME", secret_key="YOUR_SECRET_KEY")
----

=== Option 3 â€” Custom callable

// SOURCE: README â€” "3. Your own callable â€” for any other identity source"
// The callable is invoked on every request and may be sync or async.

Pass any callable that returns a bearer token string. The callable is invoked on every request, making it the natural refresh hook:

[source,python]
----
async def token_supplier() -> str:
    return await my_identity_provider.fetch_bearer()

config = Configuration(host=BASE_URL, access_token=token_supplier)
----

=== Synchronous usage

// SOURCE: README â€” "Synchronous usage â€” every method has a blocking *_sync variant"
// No event loop or await required â€” suitable for Django, Flask, scripts, notebooks.

Every async method has a blocking `*_sync` variant. No event loop or `await` is needed:

[source,python]
----
from thoughtspot_rest_api_sdk import Configuration, ThoughtSpotRestApi

config = Configuration(host=BASE_URL, access_token="YOUR_BEARER_TOKEN")
client = ThoughtSpotRestApi(configuration=config)

user = client.get_current_user_info_sync()
print(user.name)
----

This makes the SDK usable from synchronous frameworks such as Django, Flask, scripts, and Jupyter notebooks.

async with ApiClient(config) as api_client:
    users = UsersApi(api_client)
    metadata = MetadataApi(api_client)
    await users.search_users({"record_offset": 0, "record_size": 10})

=== Accessing response status and headers

// SOURCE: README â€” "Accessing response status and headers â€” *_with_http_info variant"

Each method has a `*_with_http_info` (and `*_sync_with_http_info`) variant that returns status code, headers, and deserialized data together:

[source,python]
----
response = await client.get_current_user_info_with_http_info()
print(response.status_code)
print(response.headers)
print(response.data)
----

== Streaming (SSE)

// SOURCE: README â€” Streaming section
// Endpoints that return SSE streams expose an additional *_stream async generator.

Endpoints that return server-sent event (SSE) streams expose an additional `*_stream` async generator that yields events as they arrive:

[source,python]
----
from thoughtspot_rest_api_sdk.models import SendAgentConversationMessageStreamingRequest

async for event in client.send_agent_conversation_message_streaming_stream(
    conversation_identifier="CONVERSATION_ID",
    send_agent_conversation_message_streaming_request=(
        SendAgentConversationMessageStreamingRequest(messages=["Hello"])
    ),
):
    if event.get("type") == "text-chunk":
        print(event.get("content", ""), end="", flush=True)
----

== File uploads and downloads

// SOURCE: README â€” File uploads and File downloads sections

=== File uploads

Multipart endpoints accept file content as raw bytes, a `(filename, bytes)` tuple, or an open file handle:

[source,python]
----
with open("project.zip", "rb") as f:
    await client.dbt_connection(
        connection_name="my-connection",
        database_name="MY_DB",
        import_type="ZIP_FILE",
        file_content=f,
    )
----

=== File downloads

Export endpoints return binary content as bytes. Write them to disk using `pathlib.Path`:

[source,python]
----
from pathlib import Path

data = await client.export_liveboard_report(
    metadata_identifier="LIVEBOARD_ID",
    file_format="PDF",
)
Path("report.pdf").write_bytes(data)
----

== Error handling

// SOURCE: README â€” Error handling section
// All exception classes confirmed from __init__.py exports.

API errors raise typed exceptions. Catch by HTTP status code, or catch the base `ApiException`:

[source,python]
----
from thoughtspot_rest_api_sdk.exceptions import (
    ApiException,                   # base class â€” all API errors
    BadRequestException,            # 400
    UnauthorizedException,          # 401
    ForbiddenException,             # 403
    NotFoundException,              # 404
    ConflictException,              # 409
    UnprocessableEntityException,   # 422
    ServiceException,               # 5xx
)

try:
    await client.search_metadata({"metadata": [{"type": "LIVEBOARD"}]})
except UnauthorizedException:
    # Refresh credentials and retry
    ...
except ApiException as e:
    print(e.status, e.reason, e.body, e.headers)
----

Every exception exposes `.status`, `.reason`, `.body`, `.data`, and `.headers`.

== Retries

// SOURCE: README â€” Retries section (all details confirmed)

Retries are *off by default*. Enable automatic retries with exponential backoff and jitter by setting `retries` on `Configuration`:

[source,python]
----
config = Configuration(host=BASE_URL, access_token="...", retries=3)
----

When enabled, the SDK retries on `429, 502, 503, 504` status codes and on network or timeout errors. Once the retry budget is exhausted, the final response is returned and raises the usual typed exception.

[NOTE]
====
ThoughtSpot uses POST for many read endpoints. POST requests are retried by default. To avoid retrying non-idempotent write calls, restrict the eligible methods:

[source,python]
----
config.retry_methods = {"GET", "PUT", "DELETE"}
----
====
