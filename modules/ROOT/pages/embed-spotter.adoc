  "content": "= Embed Spotter experience\n:toc: true\n:toclevels: 2\n\n:page-title: Embed Spotter\n:page-pageid: embed-spotter\n:page-description: You can use the SpotterEmbed SDK library to embed conversational analytics experience in your application.\n\nYou can embed the full Spotter experience in your applications using the `SpotterEmbed` component in the Visual Embed SDK. Using the customization settings available in the SDK for each Spotter version, you can configure the Spotter interface to suit the needs of your embedding application.\n\n== Before you begin\n\nTo embed Spotter, you need the following access and setup:\n\n* Access to a ThoughtSpot instance with the Spotter feature. If you want a specific version of Spotter enabled, work with your ThoughtSpot administrator to enable the xref:embed-ai-analytics.adoc#_feature_status_and_availability_in_embed_mode[required features and settings] on your instance.\n* Your host application domain is added to xref:security-settings.adoc[ThoughtSpot CSP and CORS allowlists].\n* Your application project has access to the xref:api-changelog.adoc[latest version of the Visual Embed SDK].\n\n[NOTE]\n====\nFor Spotter embedding that uses cookieless authentication, use Visual Embed SDK version 1.45.0 or later with ThoughtSpot Cloud version 26.2.0.cl or later. To enable proactive token refresh, set the `refreshAuthTokenOnNearExpiry` parameter to `true`. +\nSpotter workflows may run longer at times. To prevent authorization errors, set the authentication token expiration to at least 10 minutes.\n====\n\n== Import the SDK package\nImport the `SpotterEmbed` SDK library to your application environment:\n\n**npm**\n[source,JavaScript]\n----\nimport {\n    SpotterEmbed,\n    AuthType,\n    init,\n    prefetch,\n    EmbedEvent,\n    HostEvent\n}\nfrom '@thoughtspot/visual-embed-sdk';\n----\n\n**ES6**\n[source,JavaScript]\n----\n<script type = 'module'>\n    import {\n        SpotterEmbed,\n        AuthType,\n        init,\n        prefetch,\n        EmbedEvent,\n        HostEvent\n    }\nfrom 'https://cdn.jsdelivr.net/npm/@thoughtspot/visual-embed-sdk/dist/index.js';\n----\n\n== Initialize the SDK\n\nTo initialize the SDK, the following information is required:\n\n* `thoughtSpotHost` +\nThe xref:faqs.adoc#tsHostName[hostname] of your ThoughtSpot application instance.\n* `authType` +\nAuthentication type. For testing purposes, you can use `AuthType.None`. For information about other authentication options, see xref:embed-authentication.adoc[Authentication].\n\n[source,JavaScript]\n----\ninit({\n    thoughtSpotHost: 'https://your-thoughtspot-host', // Replace with your ThoughtSpot application URL\n    authType: AuthType.None, // Use the appropriate AuthType for your setup\n});\n----\n\n== Create an instance of the SpotterEmbed object\n\nCreate an instance of the `SpotterEmbed` object.\n\nIf you are embedding the Spotter Classic or Spotter 2 experience, the data source ID is required. If Spotter 3 experience is enabled on your instance, you can either specify the data source ID or enable the *Auto mode* to allow Spotter to automatically discover and select data sources.\n\n[IMPORTANT]\n====\nAuto mode is disabled by default on ThoughtSpot Embedded instances. To enable this feature on your instance, contact ThoughtSpot Support.\n====\n\n[source,JavaScript]\n----\nconst spotterEmbed = new SpotterEmbed(document.getElementById('ts-embed'), {\n    frameParams: {\n        width: '100%',\n        height: '100%',\n    },\n    worksheetId: '<%=datasourceGUID%>', // ID of the data source object. To use the Auto mode, set the ID to 'auto_mode',\n    //... other attributes\n});\n----\n\n=== Customize your embed (Optional)\nThe SDK provides configuration settings to customize the embedded Spotter interface.\nFor example, you can hide sample questions or disable data source selection using the `hideSampleQuestions` and `disableSourceSelection` parameters.\n\n[source,JavaScript]\n----\nconst conversation = new SpotterEmbed(document.getElementById('ts-embed'), {\n    //...other embed configuration attributes\n    hideSampleQuestions: true,\n    disableSourceSelection: true\n});\n----\nFor more information and examples, see xref:embed-spotter.adoc#configControls[Customizing the embedded Spotter interface].\n\n=== Register event listeners\n\nTo listen to the events emitted by the embedded ThoughtSpot component, register xref:event-embedEvents.adoc[embed event] handlers.\n\nThe following example shows how to register xref:EmbedEvent.adoc#_init[EmbedEvent.Init] and xref:EmbedEvent.adoc#_load[EmbedEvent.Load] listeners.\n\n[source,JavaScript]\n----\n// Register event listeners\nspotterEmbed.on(EmbedEvent.Init, showLoader); // Show loader when initialization starts\nspotterEmbed.on(EmbedEvent.Load, hideLoader); // Hide loader when embed is fully loaded\n----\n\n\n[source,JavaScript]\n----\n// Use EmbedEvent.Subscribed to safely trigger a HostEvent on initial load\nspotterEmbed.on(EmbedEvent.Subscribed, (eventData) => {\n    if (eventData.data.hostEventType === 'ResetSpotterConversation') {\n        spotterEmbed.trigger(HostEvent.ResetSpotterConversation);\n    }\n});\n----\n\n\nTo trigger actions on the embedded interface, use the xref:HostEvent.adoc[Host events].\n\nThe following example shows the host event to reset a Spotter conversation session:\n\n[source,JavaScript]\n----\n// Example: Add a host event to reset the Spotter conversation\ndocument.getElementById('resetBtn').addEventListener('click', () => {\n    spotterEmbed.trigger(HostEvent.ResetSpotterConversation);\n});\n----\n\n=== Render the embedded object\n\n[source,JavaScript]\n----\nspotterEmbed.render();\n----\n\n=== Verify your embed\n\n* Load the embedded object.\n** If the embedding is successful, you'll see the Spotter page.\n** Initiate a chat session, ask a question, and view the results.\n** If you see a blank screen:\n*** Verify that your embed code has the correct ThoughtSpot host URL. Ensure that your ThoughtSpot instance is accessible.\n*** Verify whether the authentication credentials used in your code are valid.\n* Verify whether the customization settings are applied.\n\n[#configControls]\n== Customizing the embedded Spotter interface\nWhen you embed Spotter, you'll notice that the embedded component loads an initial page with a prompt interface.
