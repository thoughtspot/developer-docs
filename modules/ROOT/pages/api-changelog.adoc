Analysts are configurable AI personas with a specific data context, behavioral instructions, and personality.\n\nNew configuration parameter:::\n`showAnalysts`;;\nWhen set to `true` in `SpotterEmbed`, displays the Analysts section in the embedded Spotter side navigation panel. The Analysts section is hidden by default in all embedded Spotter deployments.\n\nNew action IDs for Analysts:::\n\n* `Action.CreateAnalyst` +\nControls the visibility of the option to create a new Analyst.\n* `Action.EditAnalyst` +\nControls the visibility of the option to edit an existing Analyst.\n* `Action.MakeACopyAnalyst` +\nControls the visibility of the option to copy an Analyst.\n* `Action.ShareAnalyst` +\nControls the visibility of the option to share an Analyst with other users.\n* `Action.DeleteAnalyst` +\nControls the visibility of the option to delete an Analyst.\n\n// NOTE TO WRITER: Verify exact Action enum string values and the analystId init param name with the SDK team before publishing. (SCAL-312622)\n\n.Example: SpotterEmbed with Analysts panel enabled and create/delete actions hidden\n[source,javascript]\n----\nconst spotterEmbed = new SpotterEmbed(document.getElementById('ts-embed'), {\n  frameParams: {\n    width: '100%',\n    height: '100%',\n  },\n  worksheetId: '<%=datasourceGUID%>',\n  showAnalysts: true,\n  hiddenActions: [\n    Action.CreateAnalyst,\n    Action.DeleteAnalyst,\n  ],\n  disabledActions: [\n    Action.ShareAnalyst,\n  ],\n  disabledActionReason: 'Sharing is not available in this context.',\n});\nspotterEmbed.render();\n----\n\nFor more information, see xref:embed-spotter.adoc#spotter-analysts[Spotter Analysts in embedded apps].\n\n(SCAL-310399, SCAL-312622)\n\n|[tag greenBackground]#NEW FEATURE# a|\n\n[discrete]\n===== WYSIWYG Liveboard PDF \u2014 generally available\n\nThe continuous Liveboard PDF experience, previously available in Beta as `isContinuousLiveboardPDFEnabled`, is now generally available in 26.8.0.cl. PDF exports now accurately match the on-screen Liveboard layout.\n\nThe `Action.DownloadLiveboardAsContinuousPDF` action ID introduced in SDK 1.48.0 continues to control visibility of the new PDF download action. No configuration changes are required for existing embedded integrations \u2014 backward compatibility is preserved.\n\nFor more information, see xref:embed-pinboard.adoc#wysiwyg-pdf[Download Liveboard as PDF].\n\n(SCAL-309481)\n\n|[tag orangeBackground]#DEPRECATED# a|\n\n[discrete]\n===== Navigation V1 and V2 deprecated\n\nSDK configuration properties that set the homepage or navigation experience to V1 or V2 are deprecated and ignored starting with ThoughtSpot 26.8.0.cl. All embedded sessions render V3 navigation and homepage by default.\n\n*Migration*: Remove any `discoveryExperience` configuration that sets `primaryNavbarVersion` to a V1 or V2 value. Verify that your embed configuration, custom CSS selectors, and DOM-dependent code are compatible with the V3 navigation structure.
