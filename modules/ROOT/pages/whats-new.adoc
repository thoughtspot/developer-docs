==== V3 navigation and home page — now the default for all embedded applications [.version-badge.deprecated]#Deprecated#

Starting with 26.8.0.cl, the V3 navigation and home page experience is the only supported experience for all embedded ThoughtSpot applications. The classic (V1) experience and V2 experience modes are deprecated and are no longer rendered, regardless of any SDK flag value or cluster configuration.

If your embed configuration includes properties targeting the V1 or V2 navigation or home page experience, those settings are silently ignored from this release.

*Action required:* Update your `AppViewConfig` to use the `discoveryExperience` object with `primaryNavbarVersion: PrimaryNavbarVersion.Sliding` and `homePage: HomePage.ModularWithStylingChanges` to ensure your application renders the V3 experience correctly. For migration guidance, see xref:full-app-customize.adoc[Customize full application embedding].

// SOURCE: SCAL-312731

If you have explicitly set any of these flags to `false` in your embed configuration, your existing override is respected and no change occurs. If you have not set them, the new defaults apply automatically after upgrading to SDK v1.51.0.

==== Spotter user onboarding — starter prompts in embedded applications [.version-badge.new]#New#

Spotter now displays a guided onboarding surface for first-time users, presenting three starter categories — *Basic Search*, *Deep Analysis*, and *Data Literacy* — each populated with questions auto-generated from the user's data model.

This surface is *disabled by default* in embedded applications. Embed developers can opt in and fully customize the surface using the `starterPrompts` object in `SpotterViewConfig`:

[source,JavaScript]
----
starterPrompts: {
  enabled: true,               // Master switch — false (default) hides the whole surface
  quick: {                     // Basic Search category
    label: "",
    visibility: true,
    questions: [ { label: "", prompt: "" } ]   // Up to 4 questions
  },
  research: {                  // Deep Analysis category
    label: "",
    visibility: true,
    questions: [ { label: "", prompt: "" } ]   // Up to 4 questions
  },
  "preview-data": {            // Data Literacy — direct CTA, no questions
    label: "",
    visibility: true
  }
}
----

The `enabled` property acts as a master switch. Set it to `false` (default) to hide the entire onboarding surface. Set it to `true` to display it, with optional customization of category labels and questions.

// SOURCE: SCAL-293483, Spotter Onboarding PRD https://docs.google.com/document/d/1YQRTKvc-19GSNcbozxdQMOttm14KzMv86jhcV0y1UeQ

==== WYSIWYG Liveboard PDF — Generally Available [.version-badge.new]#New#

The continuous (WYSIWYG) Liveboard PDF feature is now generally available for embedded ThoughtSpot applications. Previously in Beta (since SDK v1.48.0), this feature is now fully supported for production use.

Set `isContinuousLiveboardPDFEnabled: true` in `LiveboardViewConfig` to render the Liveboard PDF as a single continuous page that matches the layout visible in the ThoughtSpot UI, preventing visualizations from being split across pages.

The following SDK identifiers support this feature and are now GA:

* `Action.DownloadLiveboardAsContinuousPDF`
* `EmbedEvent.DownloadLiveboardAsContinuousPDF`
* `HostEvent.DownloadLiveboardAsContinuousPDF`

For more information, see xref:embed-pinboard.adoc[Embed Liveboards].

// SOURCE: SCAL-309481

==== Liveboard CSV and XLSX downloads — GA for embedded applications [.version-badge.new]#New#

Liveboard-level CSV and XLSX downloads and scheduling are now generally available for embedded ThoughtSpot applications. The following action IDs, introduced in SDK v1.48.0 for ThoughtSpot Analytics users, are now fully supported in embedded context:

* `Action.DownloadLiveboard`
* `Action.DownloadLiveboardAsXlsx` — Downloads all table visualizations as a combined Excel workbook, with each visualization on a separate tab.
* `Action.DownloadLiveboardAsCsv` — Downloads all table visualizations as a ZIP archive of individual CSV files.

For more information, see xref:embed-pinboard.adoc[Embed Liveboards].

==== Granular download privileges [.version-badge.new]#New#

The `Can Download Data` privilege is replaced by two new, more granular privileges:

* *Can Download Visuals* — Grants permission to download visualizations as PDF or PNG.
* *Can Download Detailed Data* — Grants permission to download underlying data as XLSX or CSV.

This allows administrators to independently control who can export visual content versus raw data. Use the updated privilege type enumerations in the REST API user and group management endpoints to assign these privileges programmatically.

==== Personalized Views portability enhancements — Early Access [earlyAccess eaBackground]#Early Access#

ThoughtSpot introduces the following enhancements for Personalized Views to improve reliability and portability across SDLC deployments:

Smart merge import::
The TML import process for Liveboards now uses smart merge logic for Personalized Views. During import:
+
* Views that exist only in the destination environment are preserved.
* Views present only in the imported TML are appended.
* Views present in both the imported TML and the destination are updated.
+
This replaces the previous behavior where imports would overwrite all existing views.

`owned_by` field in TML::
The Liveboard TML schema now includes an `owned_by` field for each Personalized View, mapping the view to a specific user by username or email address. Ownership is preserved when promoting Liveboards across clusters or organizations.

`obj_id` for Personalized Views::
Personalized Views now support stable object identifiers (`obj_id`) in the TML schema and in the `/api/rest/2.0/metadata/tml/import` and `/api/rest/2.0/metadata/tml/export` endpoints, ensuring consistent identification across clusters and organizations.

For more information, see xref:tml.adoc[ThoughtSpot Modeling Language].

// SOURCE: SCAL-307283

==== Schedule attachment limit correction [.version-badge.fixed]#Fixed#

The maximum supported attachment size for ThoughtSpot communication channels (scheduled Liveboard email notifications) is *22 MB*. This value was previously documented as 25 MB. The actual limit is lower because email delivery requires base64 encoding, which increases the payload size by approximately 33%.

==== Org isolation for per-org SAML and OIDC authentication
ThoughtSpot now enforces strict org isolation when users authenticate through a per-org identity provider (IdP). When a per-org IdP sends SAML or OIDC group claims that reference Orgs outside its authorized scope, ThoughtSpot silently drops those claims and records them as security audit events. This prevents a rogue IdP administrator in one Org from using group assertions to gain unauthorized access to another Org. Manually-assigned existing Org memberships are unaffected. For more information, see xref:orgs.adoc#per-org-sso-isolation[SSO and Org isolation].

==== Chart and table overrides [.version-badge.new]#New#
You can now apply visualization overrides to charts and tables generated from a search query in ThoughtSpot search and full application embedding. The `visualOverrides` property in `SearchViewConfig` and `AppViewConfig` allows developers to apply at the embed initialization time:

* Chart overrides +
Control legend visibility and position, data label display and per-column filter
thresholds, regression lines, grid lines, axis range and label settings, series
colors, and conditional formatting rules including font and background styling.
* Table overrides +
Control column visibility, text wrapping, row height and padding density, table
theme, and column summary visibility with per-column exceptions.

For more information, see xref:viz-overrides.adoc[Configuring visualization overrides].
