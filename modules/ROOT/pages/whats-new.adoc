==== V3 navigation and home page — now the default for all embedded applications [.version-badge.deprecated]#Deprecated#

Starting with 26.8.0.cl, the V3 navigation and home page experience is the default rendering mode for all embedded ThoughtSpot applications. The classic (V1) experience and V2 experience modes are deprecated and are no longer rendered, regardless of any SDK flag value or cluster configuration.

If your embed configuration includes properties targeting the V1 or V2 navigation or home page experience, those settings are silently ignored from this release. Review your `AppViewConfig` and update your embed configuration to use the V3 `discoveryExperience` settings. For migration guidance, see xref:full-app-customize.adoc[Customize full application embedding].

// SOURCE: SCAL-312731

If you have explicitly set any of these flags to `false` in your embed configuration, your existing override is respected and no change occurs. If you have not configured them, the new defaults apply automatically after upgrading to SDK v1.51.0.

==== WYSIWYG Liveboard PDF — Generally Available [.version-badge.new]#New#

The continuous (WYSIWYG) Liveboard PDF feature is now generally available for embedded ThoughtSpot applications. Previously in Beta (since SDK v1.48.0), this feature is now fully supported for production use.

Set `isContinuousLiveboardPDFEnabled` to `true` in `LiveboardViewConfig` to enable this feature. The PDF export renders the Liveboard on a single continuous page matching the layout visible in the ThoughtSpot UI, preventing visualizations from being split across pages.

The following SDK identifiers support this feature:

* `Action.DownloadLiveboardAsContinuousPDF`
* `EmbedEvent.DownloadLiveboardAsContinuousPDF`
* `HostEvent.DownloadLiveboardAsContinuousPDF`

For more information, see xref:embed-pinboard.adoc[Embed Liveboards].

// SOURCE: SCAL-309481

==== Granular download privileges [.version-badge.new]#New#

The `Can Download Data` privilege is replaced by two new, more granular privileges:

* *Can Download Visuals* — Grants permission to download visualizations as PDF or PNG.
* *Can Download Detailed Data* — Grants permission to download underlying data as XLSX or CSV.

This change allows administrators to independently control who can export visual content versus raw data. Use the updated privilege type enumerations in the REST API user and group management endpoints to assign these privileges programmatically.

==== Spotter starter prompts in embedded applications [.version-badge.new]#New#

Embedded applications can now configure a guided onboarding experience for first-time Spotter users using the `starterPrompts` object in `SpotterViewConfig`. This feature is disabled by default for embedded applications.

The `starterPrompts` object supports the following configuration:

`enabled`:: Master switch. Set to `true` to display starter prompts to users when they open Spotter for the first time. Defaults to `false` for embedded applications.

`categories`:: Array of prompt category objects. Each category accepts:
+
* `key` — Identifier for the prompt category.
* `label` — Display label shown to the user for the category.
* `questions` — Array of suggested question strings displayed under that category.

The following example shows a basic `starterPrompts` configuration:

[source,JavaScript]
----
const embed = new SpotterEmbed("#embed", {
    starterPrompts: {
        enabled: true,
        categories: [
            {
                key: "sales",
                label: "Sales",
                questions: [
                    "What are the top 10 products by revenue?",
                    "Show me monthly sales trends"
                ]
            },
            {
                key: "operations",
                label: "Operations",
                questions: [
                    "Which regions have the highest order volume?",
                    "What is the average fulfillment time by warehouse?"
                ]
            }
        ]
    }
});
----

==== Personalized Views portability enhancements — Early Access [earlyAccess eaBackground]#Early Access#

ThoughtSpot introduces the following enhancements for Personalized Views to improve reliability and portability across SDLC deployments:

Smart merge import::
The TML import process for Liveboards now uses smart merge logic for Personalized Views. During import:
+
* Views that exist only in the destination environment are preserved.
* Views present only in the imported TML are appended.
* Views present in both the imported TML and the destination are updated.
+
This replaces the previous behavior where imports would overwrite all existing views.

`owned_by` field in TML::
The Liveboard TML schema now includes an `owned_by` field for each Personalized View. This field maps the view to a specific user by username or email address, ensuring ownership is preserved when promoting Liveboards across clusters or organizations.

`obj_id` for Personalized Views::
Personalized Views now support stable object identifiers (`obj_id`) in the TML schema and in the `/api/rest/2.0/metadata/tml/import` and `/api/rest/2.0/metadata/tml/export` API endpoints.

For more information, see xref:tml.adoc[ThoughtSpot Modeling Language].

// SOURCE: SCAL-307283

==== Schedule attachment limit correction [.version-badge.fixed]#Fixed#

The maximum supported attachment size for ThoughtSpot communication channels (scheduled Liveboard email notifications) is *22 MB*. This value was previously documented as 25 MB. The actual limit is lower because email delivery requires base64 encoding, which increases the payload size by approximately 33%.

[discrete]
==== Org isolation for per-org SAML and OIDC authentication
ThoughtSpot now enforces strict org isolation when users authenticate through a per-org identity provider (IdP). When a per-org IdP sends SAML or OIDC group claims that reference Orgs outside its authorized scope, ThoughtSpot silently drops those claims and records them as security audit events. This prevents a rogue IdP administrator in one Org from using group assertions to gain unauthorized access to another Org. Manually-assigned existing Org memberships are unaffected.

==== Chart and table overrides [.version-badge.new]#New#
You can now apply visualization overrides to charts and tables generated from a search query in ThoughtSpot search and full application embedding. The `visualOverrides` property in `SearchViewConfig` and `AppViewConfig` allows developers to apply at the embed initialization time:

* Chart overrides +
Control legend visibility and position, data label display and per-column filter
thresholds, regression lines, grid lines, axis range and label settings, series
colors, and conditional formatting rules including font and background styling.
* Table overrides +
Control column visibility, text wrapping, row height and padding density, table
theme, and column summary visibility with per-column exceptions.

For more information, see xref:viz-overrides.adoc[Configuring visualization overrides].
