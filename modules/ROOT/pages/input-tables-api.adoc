ThoughtSpot 26.8.0.cl introduces REST API v2.0 endpoints for managing input table records
programmatically. Input tables are writable tables in ThoughtSpot that allow you to push
data directly into ThoughtSpot without an external data pipeline. Use these APIs to create
new input tables, insert or update records, and delete records at scale.

[NOTE]
====
Input Tables API endpoints are available from ThoughtSpot Cloud 26.8.0.cl onwards.
====

== Supported operations

* <<create-input-table,Create an input table>> — `POST /api/rest/2.0/input-tables/create`
* <<update-input-table,Update records in an input table>> — `POST /api/rest/2.0/input-tables/{input_table_identifier}/update`
* <<delete-input-table,Delete records from an input table>> — `POST /api/rest/2.0/input-tables/{input_table_identifier}/delete`

[#create-input-table]
== Create an input table

[.version-badge.new]#New# ThoughtSpot Cloud 26.8.0.cl

Creates a new input table in ThoughtSpot with a specified schema.

// TODO: [WRITER] Confirm whether creating an input table also creates a corresponding
//       data model/worksheet automatically, or whether that step is separate.

=== Resource URL

----
POST /api/rest/2.0/input-tables/create
----

=== Request parameters

[cols="1,1,1,3", options="header"]
|===
|Parameter |Type |Required |Description

|`name`
|string
|Yes
|Name of the input table to create. Must be unique within the Org.

|`columns`
|array of objects
|Yes
|Column definitions. Each object requires: +
* `name` (string) — Column name. +
* `data_type` (string) — Data type. For example, `VARCHAR`, `INT64`, `DATE`, `BOOL`. +
* `is_primary_key` (boolean) — Whether the column is part of the primary key. Default: `false`. +
// TODO: [WRITER] Confirm the full list of supported data types with the Data platform team.
// TODO: [WRITER] Confirm whether additional column properties (nullable, default_value) are supported.

|`org_identifier`
|string
|No
|Unique ID or name of the Org. Defaults to the Org from the authentication token.

|===

=== Example request

// TODO: [WRITER] Replace with a confirmed working example from the Data platform team.
[source,cURL]
----
curl -X POST \
  'https://{ThoughtSpot-Host}/api/rest/2.0/input-tables/create' \
  -H 'Authorization: Bearer {AUTH_TOKEN}' \
  -H 'Content-Type: application/json' \
  -d '{
    "name": "sales_targets",
    "columns": [
      { "name": "region",         "data_type": "VARCHAR", "is_primary_key": true  },
      { "name": "quarter",        "data_type": "VARCHAR", "is_primary_key": true  },
      { "name": "target_revenue", "data_type": "DOUBLE",  "is_primary_key": false }
    ]
  }'
----

=== Example response

// TODO: [WRITER] Confirm the response body schema with the Data platform team.
[source,JSON]
----
{
  "input_table_id": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
  "name": "sales_targets",

=== Response codes

[cols="1,3", options="header"]
|===
|HTTP code |Description
|`200` |Input table created successfully.
|`400` |Bad request. Check required parameters and formatting.
|`401` |Unauthorized. Authentication token is missing or invalid.
|`403` |Forbidden. Insufficient privilege to create input tables.
|`409` |Conflict. An input table with this name already exists in the Org.
|`500` |Internal server error. Contact ThoughtSpot Support.
|===

[#update-input-table]
== Update records in an input table

[.version-badge.new]#New# ThoughtSpot Cloud 26.8.0.cl

Inserts new records or updates existing records in a specified input table (upsert behavior).
Records matching the primary key are updated; unmatched records are inserted.

// TODO: [WRITER] Replace with a confirmed working example from the Data platform team.
[source,cURL]
----

  'https://{ThoughtSpot-Host}/api/rest/2.0/input-tables/sales_targets/update' \
