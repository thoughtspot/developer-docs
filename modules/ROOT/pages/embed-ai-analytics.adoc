= Embed AI Search and Analytics
:toc: true
:toclevels: 3

:page-title: Embed AI Search and Analytics
:page-pageid: embed-ai-search-analytics
:page-description: To embed the ThoughtSpot Spotter and conversational analytics experience in your app, you can use the `SpotterEmbed` or `SpotterAgentEmbed` SDK components.

Spotter provides an interactive AI-powered search and conversational analytics experience for ThoughtSpot application users. It empowers users to ask questions in natural language and analyzes data to generate business-ready insights instantly.

ThoughtSpot supports three experience modes: Spotter Classic, Spotter Agent (Spotter 2), and Spotter 3. Spotter 3 supports several new features, agentic analytics, and provides an enhanced user experience.

== SDK libraries for embedding Spotter
ThoughtSpot provides the following Visual Embed SDK libraries for embedding Spotter capabilities in your app:

* xref:embed-spotter.adoc[`SpotterEmbed`] +
Use this SDK component to xref:embed-spotter.adoc[embed the full Spotter experience] in your app. The features in a Spotter embed vary based on the Spotter version.

* xref:embed-spotter-agent.adoc[`SpotterAgentEmbed`] +
Also known as `bodyless` embedding, `SpotterAgentEmbed` allows you to integrate Spotter capabilities into your own applications or chatbot, route user questions to ThoughtSpot, and receive structured answers and visualizations. Unlike the full Spotter experience, the "bodyless" embed includes only a prompt panel with no additional buttons or selectors.

== Spotter Analysts (Spots) in embedded applications [earlyAccess eaBackground]#Early Access#

Spotter Analysts, also referred to as *Spots*, are custom AI analysts that can be scoped to a specific data domain and surfaced to end users in the Spotter interface. Each Spot has a defined name, optional custom icon, a set of allowed data models, and behavioral instructions that shape how it responds to user queries.

=== Prerequisites

* Spotter 3 must be enabled on your ThoughtSpot instance.
* An administrator must enable the Spotter Analysts feature in the Admin console: *Spotter* > *Capabilities* > *Enable Spotter Analysts*. This setting is org-aware.
* The Spot must be created and configured by an administrator before it can be surfaced in an embedded application.

=== Default behavior in embedded Spotter

When you embed Spotter using `SpotterEmbed`, the following defaults apply for Spotter Analysts:

* *Analysts are disabled by default in embed.* The Analysts section on the side navigation panel is not shown.
* Users cannot create, edit, copy, share, or delete Analysts from within the embedded experience unless those actions are explicitly enabled.

This default ensures that embedding applications are unaffected by the Analysts feature until you opt in.

=== Enabling and controlling Analysts in embed

Use the `customizationConfig` property in `SpotterEmbed` to control the Analysts experience in your application.

==== Show the Analysts panel

To display the Analysts section in the embedded Spotter side navigation:

=== Upgrade behavior

When upgrading to ThoughtSpot Cloud 26.8.0.cl:

* Existing embedded Spotter integrations are unaffected. Analysts are not shown unless you explicitly enable them.
* If a user's access to a data model is revoked after a Spot was created, the Spot returns an error when accessed. Users cannot make a copy of a Spot they no longer have data access for.
* If a Spot is deleted, any saved conversations associated with that Spot are no longer accessible via the deleted Spot's context. Users cannot ask follow-up questions in those conversations.
